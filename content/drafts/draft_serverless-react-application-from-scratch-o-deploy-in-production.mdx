Neste artigo, vamos ingressar em uma jornada onde faremos do zero uma aplicação em React até realizar o seu deploy em produção. Não somente, vamos alinhar alguns pontos sobre quais ferramentas podem ser úteis para analisar e monitorar a saúde da sua aplicação quando ela estiver pronta para uso.


Para isso, vamos criar um projeto real.


### Sobre o Projeto


Este é um projeto que já venho pensando em construir há algum tempo e após toda turbulência e enxurradas de entrevistas devido a um Layoff que passei. 

Sim, fui um dos impactados. Não só eu, todo meu time. Triste, né? Mas, a vida segue e depois da tempestade vem a bonança. O importante é seguir firme e forte!
Voltando pro que importa!


O projeto será open-source e nele teremos algumas regras de negócios bem definidas. 


Vamos trazer o que há de mais moderno para esta aplicação. Não somente, traremos a Arquitetura Serverless para jogar no nosso time e facilitar a construção da Infraestrutura, Banco de Dados e API. 


A ideia deste projeto é reunir uma série de conteúdo técnicos direcionados para entrevistas técnicas.


Ou seja, se você está participando de um processo seletivo, entende que este momento é uma etapa onde o seu conhecimento técnico será avaliado, mas não apenas isso, a sua maneira como lidar com a solução dos problemas também.


Para isso, é importante que você esteja confiante e saiba a necessidade que aquela oportunidade para a qual você se candidatou está pedindo.


É importante ficar atento ao que a vaga pede pois algumas perguntas técnicas podem estar relacionadas diretamente ao problemas que necessitam de solução. Desta forma, não se atendo à perguntas diretas sobre funcionalidades ou recursos de uma ferramenta ou linguagem, ou mesmo aqueles testes de raciocínio lógico com Algoritmos e Estrutura de Dados.
