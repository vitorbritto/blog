O Virtual DOM é uma das principais features do React. É um objeto JavaScript que basicamente é uma abstração do DOM real e uma das peças centrais para a alta performance da aplicação.

Neste artigo, vamos cobrir o Virtual DOM, explorando seus benefícios em aplicações React.

## Um breve contexto sobre o DOM

Para entender o Virtual DOM e descobrir por que o React o implementa, vamos dar uma olhada em como o DOM real do browser funciona.

Quando um usuário faz uma requisição de uma página web, o browser recebe do servidor um documento HTML correspondente. A engine do browser então constrói uma estrutura em árvore a partir do HTML para mostrar ao usuário a página solicitada no client.

Essa estrutura em árvore é chamada de Document Object Model, também conhecida como DOM. É uma representação estrutural do documento web em forma de nodes e objetos.

O DOM serve como uma interface para o documento web, de modo que o JavaScript possa acessar, manipular e interagir programaticamente com o conteúdo do documento. Por exemplo, os desenvolvedores podem usar APIs do DOM para adicionar ou remover elementos, modificar sua aparência e executar ações de usuário em elementos web.

## Os impactos de Repaint e Reflow

Mudar qualquer informação que tenha sido usada para construir a render tree pode resultar em uma das condições abaixo (ou nas duas).

- Reflow: quando partes da render tree (ou a árvore inteira) precisam ser revalidadas e as dimensões dos nodes recalculadas.
- Repaint: quando partes da tela precisam ser atualizadas, seja por mudanças nas propriedades geométricas de um node, seja por mudanças estilísticas, como alterar a cor de fundo.

### Reflow

Reflow significa recalcular as posições e geometrias dos elementos no documento, com o propósito de re-renderizar parte ou todo o documento. Como o reflow é uma operação que bloqueia o usuário no browser, é útil os desenvolvedores entenderem como melhorar o tempo de reflow e também os efeitos de várias propriedades do documento (profundidade do DOM, eficiência de regras CSS, diferentes tipos de mudanças de estilo) sobre o tempo de reflow. Às vezes, fazer reflow de um único elemento no documento pode exigir reflow dos seus elementos pais e também de quaisquer elementos que o sigam.

### Repaint

Repaint significa repintar o elemento na tela quando a aparência do elemento muda, afetando a visibilidade dele, mas não afetando o layout.

Coisas que disparariam um repaint:

- mudar a visibilidade de um elemento;
- mudar o outline do elemento;
- mudar a cor de fundo.

## Virtual DOM ao resgate

A estratégia de reduzir e agrupar mudanças no DOM, levada a outro nível de abstração, é a ideia por trás do Virtual DOM do React.

O React não está realmente fazendo nada novo. É só uma jogada estratégica. Ele guarda uma réplica do DOM real em memória. Quando você modifica o DOM, o React aplica primeiro essas mudanças no DOM em memória e, usando seu diffing algorithm, descobre o que realmente mudou.

Por fim, ele agrupa as mudanças e as aplica no DOM real de uma só vez. Assim, minimizando o reflow e o repaint.

Essencialmente, o virtual DOM oferece um mecanismo que permite ao DOM real computar operações mínimas no DOM ao re-renderizar a UI.

Ele permite que você atualize uma view sempre que o state muda, criando uma representação completa da view e então fazendo patch no DOM de forma eficiente para que ele fique exatamente como você descreveu.

Por exemplo, quando um elemento no DOM real é alterado, o DOM vai re-renderizar o elemento e todos os seus filhos. Quando se trata de construir aplicações web complexas com bastante interatividade e mudanças de state, essa abordagem é lenta e ineficiente.

Em vez disso, no processo de renderização, o React usa o conceito de virtual DOM, que se alinha com sua abordagem declarativa. Assim, podemos especificar em qual state queremos que a UI esteja, e depois o React faz isso acontecer.

## Como o React implementa o Virtual DOM

Quando renderizamos a interface do usuário de uma aplicação, o React cria uma virtual DOM tree representando aquela UI e a guarda em memória. Na próxima atualização, quando os dados que renderizam o app mudam, o React vai automaticamente criar uma nova virtual DOM tree para a atualização.

![Virtual DOM](/blog/articles/img_69_2.png)

Depois que o React cria a nova virtual DOM tree, ele a compara com o snapshot anterior usando um diffing algorithm chamado reconciliation para descobrir quais mudanças são necessárias.

Após o processo de reconciliation, o React usa uma renderer library como o ReactDOM, que pega as informações do diff para atualizar o app renderizado. Essa biblioteca garante que o DOM real só receba e repinte o node ou nodes atualizados.

Em resumo, a cada render, o React compara a virtual DOM tree com a versão anterior para determinar qual node será atualizado, garantindo que o node atualizado esteja em sintonia com o DOM real.

## Conclusão

O React usa o virtual DOM como uma estratégia para computar operações mínimas no DOM ao re-renderizar a UI.

O virtual DOM oferece um mecanismo que abstrai manipulações manuais do DOM, longe do desenvolvedor, ajudando a escrever um código mais previsível. Ele faz isso comparando duas render trees para determinar exatamente o que mudou, atualizando apenas o necessário no DOM real.
