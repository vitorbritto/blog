Os React hooks, especialmente quando combinados com componentes funcionais, trouxeram uma mudança revolucionária na forma como trabalhamos com funcionalidades comuns do React, como props, state, context, refs e lifecycle methods. Embora usar os hooks padrão seja satisfatório, o verdadeiro divisor de águas vem com a introdução dos custom hooks. Imagine que você criou um pedaço de lógica para um componente usando hooks básicos como `useEffect` e `useState`. Mais tarde, percebe a necessidade de reutilizar essa lógica em um novo componente. Embora copiar e colar pareça uma solução direta, usar custom hooks para o mesmo propósito não é só mais agradável, como também oferece várias vantagens. Ao encapsular lógicas frequentemente usadas em um custom hook, seu código fica mais limpo, mais reutilizável e, claro, mais fácil de manter.

## O antes e depois

Vamos começar com um exemplo de fetch de dados de uma API sem usar custom hook e depois refatorar usando um custom hook.

```js
import React, { useState, useEffect } from "react";

const DataFetchingWithoutCustomHook = () => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(
          "https://jsonplaceholder.typicode.com/todos/1"
        );
        const result = await response.json();
        setData(result);
      } catch (error) {
        setError(error);
      } finally {
        setLoading(false);
      }
    };
    fetchData();
  }, []);

  if (loading) {
    return <p>Loading...</p>;
  }

  if (error) {
    return <p>Error: {error.message}</p>;
  }

  return (
    <div>
      <h1>Data: {data.title}</h1>
    </div>
  );
};

export default DataFetchingWithoutCustomHook;
```

E então, como encapsular essa lógica em um custom hook:

```js
import React, { useState, useEffect } from "react";
// Custom Hook para Data Fetching
const useDataFetching = (url) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        const result = await response.json();
        setData(result);
      } catch (error) {
        setError(error);
      } finally {
        setLoading(false);
      }
    };
    fetchData();
  }, [url]);

  return { data, loading, error };
};

const DataFetchingWithCustomHook = () => {
  const { data, loading, error } = useDataFetching(
    "https://jsonplaceholder.typicode.com/todos/1"
  );

  if (loading) {
    return <p>Loading...</p>;
  }

  if (error) {
    return <p>Error: {error.message}</p>;
  }

  return (
    <div>
      <h1>Data: {data.title}</h1>
    </div>
  );
};

export default DataFetchingWithCustomHook;
```

## Conclusão

O custom hook pattern no React se mostra uma solução poderosa e elegante para encapsular e reutilizar lógica entre componentes. Ao extrair funcionalidades comuns para custom hooks, os desenvolvedores podem aprimorar a organização, manutenibilidade e reusabilidade do código. No exemplo apresentado, o custom hook para data fetching simplifica a lógica do componente, deixando-o mais focado e mais fácil de entender. Esse pattern se alinha com a filosofia de composabilidade do React, permitindo que os desenvolvedores construam codebases modulares e manuteníveis enquanto promovem o reuso eficiente de lógica em suas aplicações. No geral, o custom hook pattern contribui para uma experiência de desenvolvimento mais simplificada e modular no React.
