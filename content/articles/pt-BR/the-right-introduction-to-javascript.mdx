JavaScript é uma linguagem de programação de alto nível, interpretada e multi-paradigma. Isso significa que o código JavaScript é escrito em um formato legível por humanos e executado por um interpretador em runtime, permitindo que rode em diferentes ambientes, como browsers, servers e até dispositivos mobile.

Em browsers, o JavaScript é primariamente usado para tornar páginas web interativas, controlando o comportamento do conteúdo HTML e CSS, manipulando o Document Object Model (DOM) e respondendo a eventos do usuário.

## Contexto Histórico do JavaScript

O JavaScript foi criado por Brendan Eich em 1995 enquanto trabalhava na Netscape Communications Corporation. A linguagem foi originalmente nomeada "Mocha" e depois "LiveScript" antes de ser renomeada para "JavaScript", capitalizando a popularidade da linguagem de programação Java.

O JavaScript foi projetado para ser uma linguagem de programação leve e fácil de usar para adicionar interatividade a páginas web. No entanto, ao longo dos anos, a linguagem evoluiu para se tornar uma das linguagens de programação mais populares e versáteis, usada em uma ampla variedade de aplicações, incluindo desenvolvimento web, servers, apps mobile e até aplicações desktop.

## Principais features do JavaScript

### Tipos de Dados Dinâmicos

JavaScript é dinamicamente tipado, o que significa que os tipos de dados são checados em runtime, permitindo que variáveis mudem de tipo ao longo do tempo.

### First-Class Functions

Funções no JavaScript são cidadãs de primeira classe, o que significa que podem ser atribuídas a variáveis, passadas como argumentos e retornadas de outras funções.

### Orientação a Objetos baseada em Prototype

JavaScript usa um modelo de objetos baseado em prototype em que objetos podem herdar propriedades e métodos uns dos outros.

### Eventos e Assincronicidade

JavaScript permite tratar eventos de forma assíncrona, crucial para a interatividade em aplicações web.

### Manipulação do DOM

JavaScript é amplamente usado para manipular o Document Object Model (DOM) em browsers, permitindo aos desenvolvedores criar páginas web dinâmicas e interativas.

### Regular Expressions

JavaScript suporta regular expressions, que são patterns de texto usados para buscar e manipular strings.

### Browser APIs

JavaScript tem APIs nativas que permitem ao código interagir com o ambiente do browser, incluindo APIs para manipular o DOM, fazer HTTP requests, armazenar dados localmente e mais.

### Modules e Packages

Com a introdução dos modules do ES6, o JavaScript agora suporta nativamente importar e exportar modules, permitindo aos desenvolvedores organizar e reutilizar código de forma modular.

### Promises e Async/Await

JavaScript suporta promises e async/await, mecanismos para lidar com operações assíncronas de uma maneira mais legível e eficiente.

### Extensões e Bibliotecas

JavaScript tem uma ampla variedade de extensões e bibliotecas que estendem suas capacidades e facilitam o desenvolvimento de aplicações complexas.

JavaScript suporta vários paradigmas de programação.

### Programação Orientada a Objetos

JavaScript usa um modelo orientado a objetos baseado em prototype, em que objetos podem herdar propriedades e métodos uns dos outros.

```js
// Classe representando um Rectangleclass Rectangle {  constructor(width, height) {    this.width = width;    this.height = height;  }  // Método para calcular a área do retângulo  calculateArea() {    return this.width * this.height;  }  // Método para calcular o perímetro do retângulo  calculatePerimeter() {    return 2 * (this.width + this.height);  }}// Cria um novo objeto Rectangleconst rectangle1 = new Rectangle(5, 3);// Calcula e exibe a áreaconsole.log("Area of rectangle1:", rectangle1.calculateArea());// Calcula e exibe o perímetroconsole.log("Perimeter of rectangle1:", rectangle1.calculatePerimeter());
```

Neste exemplo, definimos uma classe `Rectangle` com propriedades para width e height, bem como métodos para calcular a área e o perímetro do retângulo. Em seguida, criamos uma instância da classe `Rectangle` chamada `rectangle1` com largura de 5 unidades e altura de 3 unidades. Por fim, usamos os métodos `calculateArea()` e `calculatePerimeter()` para computar e exibir a área e o perímetro do retângulo, respectivamente. Isso demonstra os princípios básicos da programação orientada a objetos no JavaScript, incluindo encapsulamento, abstração, herança e polimorfismo.

### Programação Funcional

JavaScript suporta programação funcional, em que funções são tratadas como cidadãs de primeira classe e podem ser passadas como argumentos e retornadas de outras funções.

```js
// Array de númerosconst numbers = [1, 2, 3, 4, 5];// Função para calcular a soma de um arrayconst sum = array => array.reduce((acc, current) => acc + current, 0);// Função para calcular o produto de um arrayconst product = array => array.reduce((acc, current) => acc * current, 1);// Função para filtrar números pares de um arrayconst filterEven = array => array.filter(num => num % 2 === 0);// Função para mapear cada elemento de um array para seu quadradoconst mapToSquare = array => array.map(num => num * num);// Calcula e exibe a soma dos númerosconsole.log("Sum of numbers:", sum(numbers));// Calcula e exibe o produto dos númerosconsole.log("Product of numbers:", product(numbers));// Filtra os números pares e exibe o resultadoconsole.log("Even numbers:", filterEven(numbers));// Mapeia os números para seus quadrados e exibe o resultadoconsole.log("Numbers squared:", mapToSquare(numbers));
```

Neste exemplo, definimos várias funções que seguem os princípios da programação funcional:

1. `sum`: usa a higher-order function `reduce` para calcular a soma de um array.
2. `product`: usa a higher-order function `reduce` para calcular o produto de um array.
3. `filterEven`: usa a higher-order function `filter` para filtrar os números pares de um array.
4. `mapToSquare`: usa a higher-order function `map` para mapear cada elemento de um array para seu quadrado.

Essas funções são pure functions, o que significa que não mutam o state e produzem o mesmo output para o mesmo input. Demonstramos o uso dessas funções com o array `numbers`, mostrando como conceitos de programação funcional podem ser aplicados no JavaScript para realizar operações comuns em arrays.

### Programação Event-Driven

JavaScript é amplamente usado para programação event-driven em browsers, em que os programas respondem a eventos do usuário, como cliques de mouse e pressionamentos de tecla.

```js
<!DOCTYPE html><html lang="en"><head>  <meta charset="UTF-8">  <meta name="viewport" content="width=device-width, initial-scale=1.0">  <title>Event-Driven Programming Example</title></head><body>  <!-- Elemento de botão -->  <button id="myButton">Click Me!</button>  <script>    // Função para tratar o evento de click do botão    function handleClick() {      console.log("Button clicked!");    }    // Pega referência para o elemento de botão    const button = document.getElementById('myButton');    // Adiciona event listener para o evento de click    button.addEventListener('click', handleClick);  </script></body></html>
```

Neste exemplo:

1. Temos um elemento de botão com o id `myButton`.
2. Definimos uma função JavaScript `handleClick()` que loga uma mensagem no console quando chamada.
3. Usamos `document.getElementById()` para obter uma referência ao elemento de botão.
4. Usamos `addEventListener()` para adicionar um event listener de click ao botão. Quando o botão é clicado, a função `handleClick()` será chamada.

Isso demonstra a natureza event-driven do JavaScript, em que ações do usuário (clicar no botão) disparam funções para serem executadas em resposta a esses eventos. Programação event-driven é fundamental para construir aplicações web interativas e dinâmicas.

### Programação Assíncrona

JavaScript é frequentemente usado para programação assíncrona, em que operações são realizadas de maneira não-bloqueante, permitindo que o código continue executando enquanto espera pela conclusão das operações assíncronas.

```js
// Função simulando uma operação assíncrona com delayfunction simulateAsyncOperation() {  return new Promise(resolve => {    setTimeout(() => {      resolve("Async operation completed");    }, 2000); // Simula um delay de 2 segundos  });}// Função async usando async/await para realizar operação assíncronaasync function doAsyncTask() {  console.log("Starting async task...");    try {    const result = await simulateAsyncOperation();    console.log(result);    console.log("Async task completed");  } catch (error) {    console.error("Error:", error);  }}// Chama a função asyncdoAsyncTask();console.log("Async task initiated");
```

Neste exemplo:

1. Definimos uma função `simulateAsyncOperation()` que retorna uma Promise, simulando uma operação assíncrona com um delay de 2 segundos.
2. Definimos uma função async `doAsyncTask()` que usa a keyword `async` antes da declaração da função. Dentro dessa função, usamos `await` para esperar pela conclusão da operação assíncrona.
3. Chamamos a função `doAsyncTask()`, que inicia a operação assíncrona. Enquanto a operação assíncrona está em andamento, outras tarefas síncronas podem continuar executando.
4. Quando a operação assíncrona é concluída, o valor resolvido é logado no console.

Isso demonstra como a sintaxe async/await nos permite escrever código assíncrono de uma forma mais semelhante a código síncrono, facilitando o entendimento e a manutenção.

### Programação Baseada em Prototype

JavaScript usa um modelo orientado a objetos baseado em prototype, em que objetos podem herdar propriedades e métodos uns dos outros.

```js
// Define um objeto prototypeconst animalPrototype = {  // Método para fazer o som do animal  makeSound() {    console.log(`${this.name} says ${this.sound}`);  }};// Cria um novo objeto baseado no prototypeconst dog = Object.create(animalPrototype);dog.name = "Dog";dog.sound = "Woof";// Cria outro objeto baseado no prototypeconst cat = Object.create(animalPrototype);cat.name = "Cat";cat.sound = "Meow";// Chama o método makeSound nos objetos dog e catdog.makeSound(); // Output: Dog says Woofcat.makeSound(); // Output: Cat says Meow
```

Neste exemplo:

1. Definimos um objeto prototype `animalPrototype` contendo um método `makeSound`.
2. Criamos os objetos `dog` e `cat` usando o método `Object.create()`, especificando `animalPrototype` como o prototype.
3. Adicionamos propriedades específicas `name` e `sound` a cada objeto.
4. Tanto o objeto `dog` quanto o `cat` herdam o método `makeSound` do objeto `animalPrototype` e, quando chamados, exibem o nome e o som dos respectivos animais.

Isso demonstra como objetos no JavaScript podem herdar propriedades e métodos de objetos prototype, formando uma prototype chain. A programação baseada em prototype permite criação flexível de objetos e herança no JavaScript.

Essa flexibilidade em suportar vários paradigmas de programação é uma das razões pelas quais o JavaScript é tão popular e versátil.

## Ciclo de vida do código JavaScript

JavaScript é executado em um ambiente de runtime, que pode ser um browser, um server ou um ambiente de execução específico, como o Node.js.

O ambiente de runtime oferece um interpretador que lê e executa o código JavaScript, bem como APIs (Application Programming Interfaces) que permitem ao código JavaScript interagir com o ambiente em que está rodando.

Ao longo desse processo, o interpretador também cuida do gerenciamento de memória, alocação de recursos e tratamento de exceções, garantindo que o código JavaScript seja executado de forma eficiente e segura.

O interpretador JavaScript é uma parte importante do ambiente de execução da linguagem. É responsável por traduzir o código JavaScript em instruções compreensíveis pelo computador e executá-las.

Deixa eu explicar o processo passo a passo.

### Análise Lexical (Tokenisation)

O primeiro passo é a análise lexical, em que o interpretador divide o código-fonte em tokens. Tokens são unidades lexicais, como keywords (`if`, `for`, `function`), identificadores (nomes de variáveis e funções), operadores (`+`, `-`, `*`, etc.) e literais (números, strings, etc.).

### Contexto Histórico

A análise lexical tem suas raízes nos primeiros dias da computação e das linguagens de programação. Nos primeiros compiladores, a análise lexical era realizada manualmente, em que os programadores tinham que analisar o código-fonte em busca de patterns específicos e converter manualmente esses patterns em tokens. Isso era tedioso e propenso a erros.

Com o avanço da teoria de compiladores e das linguagens de programação, surgiram técnicas automatizadas para análise lexical. O conceito de regular expressions, proposto por Stephen Kleene nos anos 1950, teve um papel crucial no desenvolvimento de algoritmos eficientes para análise lexical.

### Como a Análise Lexical Funciona

A análise lexical é o processo de converter sequências de caracteres em tokens significativos para o compilador ou interpretador. Isso envolve dividir o código-fonte em unidades lexicais ou tokens, que são as menores unidades reconhecíveis da linguagem de programação.

O processo de análise lexical tipicamente segue estes passos:

1. Scanning: o código-fonte é escaneado caractere por caractere. O lexer identifica patterns específicos, como keywords, identificadores, operadores e símbolos, usando regular expressions ou regras de análise pré-definidas.
2. Reconhecimento de Tokens: com base nos patterns identificados durante o scanning, o lexer gera os tokens correspondentes. Cada token é uma estrutura de dados que contém informações sobre o tipo do token e, às vezes, seu valor.
3. Descarte de Whitespace e Comentários: o lexer tipicamente ignora whitespace e comentários, já que não têm significado semântico na linguagem de programação.

### Tokens no JavaScript

O conjunto completo de tokens no JavaScript inclui keywords, identificadores, literais, operadores e pontuação. Aqui está uma lista geral dos principais tokens no JavaScript:

- Keywords: `if`, `else`, `for`, `while`, `function`, `var`, `let`, `const`, `return`, `switch`, `case`, `default`, etc.
- Identificadores: nomes usados para identificar variáveis, funções, classes e outros elementos no código. Devem começar com uma letra, underscore `_` ou cifrão `$`, seguidos por letras, dígitos, underscores ou cifrões. Por exemplo: `foo`, `_bar`, `$baz`, `javascript`, etc.
- Literais: representam valores de dados fixos. Podem ser numéricos, strings, booleans, arrays ou objetos.
- Operadores: símbolos que realizam operações em operandos. Por exemplo: `+`, `=`, `!==`, `&&`, `||`, `typeof`, `instanceof`, etc.
- Pontuação: símbolos de pontuação usados para separar tokens e definir a estrutura do código. Por exemplo: `(`, `)`, `[`, `]`, `;`, `:`, `.`, etc.

Tenha em mente que este é um resumo geral, e pode haver algumas variações ou tokens específicos dependendo da versão do ECMAScript (a especificação padrão do JavaScript) e das extensões da linguagem em diferentes ambientes de execução (como browsers ou ambientes de server).

### Análise Sintática (Parsing)

Em seguida, o interpretador analisa a estrutura do código usando regras gramaticais definidas pela especificação da linguagem. Essa etapa cria uma Abstract Syntax Tree (AST), que representa a estrutura hierárquica do código. Por exemplo, em uma expressão `a + b`, a AST teria um node para a operação de adição com dois filhos representando as variáveis `a` e `b`.

### O que é uma AST?

Uma Abstract Syntax Tree (AST) é uma estrutura de dados frequentemente usada em compiladores e interpretadores para representar a estrutura de um programa de forma abstrata, ou seja, sem incluir detalhes de implementação como ponteiros de memória ou tokens específicos da linguagem.

### Como Ela é Construída?

Uma AST é construída a partir da análise sintática do código-fonte. Durante essa análise, o código é parseado e quebrado em uma estrutura de dados em forma de árvore, em que cada node representa uma construção sintática da linguagem de programação, como instruções, expressões, operadores, etc.

### Quais são os Nodes de uma AST?

Os nodes de uma AST representam os elementos estruturais do código-fonte. Podem incluir nodes para instruções (como declarações de variáveis, declarações de função, classes), expressões (como operações aritméticas, function calls, expressões condicionais), operadores, identificadores (nomes de variáveis, nomes de funções, etc.) e assim por diante.

  return a + b;

const result = sum(2, 5);

## Geração de Código

Dependendo da implementação do interpretador, o próximo passo pode variar:

- Pure Interpretation
- Just-In-Time (JIT) Compilation

Vamos entender mais sobre eles.

Pure Interpretation
Em pure interpretation, o interpretador percorre a AST diretamente, executando as instruções uma por uma. Isso significa que ele lê cada node da AST e executa a operação correspondente em tempo real.

Por exemplo, considere o seguinte código JavaScript:

```js
function sum(a, b) {
  return a + b;
}
const result = sum(3, 4);
console.log(result);
```

Durante a interpretação, o interpretador percorreria a AST gerada para esse código, executando as operações correspondentes para cada node. Por exemplo, executaria a declaração da função `sum`, a chamada da função `sum` com os argumentos 3 e 4, e a chamada da função `console.log` para exibir o resultado.

### Just-In-Time (JIT) Compilation

Em um ambiente de JIT compilation, o código JavaScript é inicialmente interpretado, mas partes do código que são executadas com frequência são compiladas em código de máquina nativo para melhorar a performance. Isso é feito dinamicamente durante a execução do programa.

Por exemplo, o interpretador V8 no Chrome usa uma abordagem de JIT compilation para melhorar a performance do código JavaScript. Durante a execução do programa, o V8 monitora quais partes do código são executadas com frequência e, se necessário, compila essas partes em código de máquina nativo para melhorar a performance.

### Execução

Por fim, o código é executado de acordo com as instruções fornecidas pelo interpretador. Isso inclui o tratamento de variáveis, function calls, controle de fluxo e todas as outras operações definidas pelo código JavaScript.

Ao longo desse processo, o interpretador também cuida do gerenciamento de memória, alocação de recursos e tratamento de exceções, garantindo que o código JavaScript seja executado de forma eficiente e segura.

O Execution Context
O Execution Context do JavaScript é um conceito fundamental para entender como o código JavaScript é executado. Essencialmente, refere-se ao ambiente em que o código JavaScript é avaliado e executado. Toda vez que um script roda no JavaScript, ele opera dentro de um execution context.

Existem dois tipos principais de execution contexts no JavaScript:

- Global Execution Context: este é o execution context default ou mais externo. Representa o ambiente em que o código fora de qualquer função é executado. Em um browser, o global execution context é associado ao objeto `window`.
- Function Execution Context: cada vez que uma função é invocada, um novo execution context é criado especificamente para essa função. Esse contexto inclui os argumentos da função, variáveis locais e quaisquer funções aninhadas.

Há um terceiro tipo de execução, mas, pela segurança do seu código, evite-o. É o Eval Execution Context.
Os execution contexts são criados e gerenciados pela engine JavaScript durante a execução do código. Eles formam uma stack conhecida como "execution stack" ou "call stack".

A Call Stack será coberta em breve em um próximo artigo. Também vou escrever sobre Event Loop e Call-back Queue. 🚀
Quando uma função é invocada, um novo execution context para essa função é empilhado na stack. Assim que a função completa sua execução, seu contexto é removido da stack, e o controle retorna para o execution context anterior.

Cada execution context tem duas fases: a Creation Phase e a Execution Phase.

Para entender essas fases, vamos começar com o seguinte exemplo:

```js
const source = 10; const timesTen = (num) => num * 10const result = timesTen(source); console.log(result); // 100
```

Neste exemplo:

- Primeiro, declara a variável `source` e inicializa seu valor com 10.
- Segundo, declara a função `timesTen()` que aceita um argumento e retorna um valor que é o resultado da multiplicação do argumento por 10.
- Terceiro, chama a função `timesTen()` com o argumento sendo o valor da variável `source` e armazena o resultado na variável `result`.
- Por fim, imprime a variável `result` no Console.

### A Creation Phase

Quando a engine JavaScript executa um script pela primeira vez, ela cria o global execution context.

Global Execution Context — Creation PhaseFunction Execution Context — Creation Phase
Durante essa fase, a engine JavaScript realiza as seguintes tarefas:

- Cria o objeto global, ou seja, `window` no browser ou `global` no Node.js.
- Cria o objeto `this` e o vincula ao objeto global.
- Configura uma memory heap para armazenar variáveis e referências de função.
- Armazena as declarações de função na memory heap e as variáveis dentro do global execution context com os valores iniciais como `undefined`.

Quando a engine JavaScript executa o exemplo de código acima, ela faz o seguinte na creation phase:

- Primeiro, armazena as variáveis `source` e `result` e a declaração de função `timesTen()` no global execution context.
- Segundo, inicializa as variáveis `source` e `result` como `undefined`.

Após a creation phase, o global execution context passa para a execution phase.

### Gerenciamento de Memória

No JavaScript, o gerenciamento de memória é automatizado e baseado em garbage collection. Isso significa que os desenvolvedores não precisam se preocupar explicitamente com alocar ou desalocar memória, como em linguagens de baixo nível. Em vez disso, o mecanismo do garbage collector cuida disso automaticamente.

Quando um programa JavaScript é executado, ele aloca espaço de memória conforme necessário para armazenar variáveis, objetos, funções e outros dados. Conforme o código roda, o mecanismo de garbage collection rastreia referências para determinar quais objetos ainda estão sendo usados pelo programa. Se um objeto não tem mais referências apontando para ele, significa que não é mais acessível e pode ser considerado lixo. Nesse momento, o garbage collector entra em ação e libera a memória ocupada por esse objeto, tornando-a disponível para reuso para outros propósitos.

JavaScript usa uma estratégia de garbage collection chamada "incremental garbage collection", que divide o processo de garbage collection em pequenos passos para minimizar o impacto na performance do programa. Além disso, diferentes browsers e ambientes de execução podem ter implementações específicas de garbage collection, com diferentes estratégias e algoritmos para otimizar o processo.

Apesar do gerenciamento automatizado de memória, os desenvolvedores ainda precisam estar cientes de potenciais memory leaks. Um memory leak ocorre quando um objeto que não é mais necessário ainda ocupa memória porque ainda é referenciado por outras partes do código. Isso pode acontecer, por exemplo, quando há referências circulares entre objetos ou quando eventos não são devidamente desinscritos.

### Alocação de Recursos

No JavaScript, assim como o gerenciamento de memória, a alocação de recursos é tratada automaticamente na maioria dos casos. Os recursos mais comuns que precisam ser alocados e liberados são conexões de rede, file handlers, conexões de banco de dados e outros recursos externos.

A maioria das operações que envolvem alocação de recursos no JavaScript é feita de forma assíncrona e não-bloqueante, o que significa que o código pode continuar executando enquanto espera o recurso ficar disponível. Por exemplo, ao fazer uma HTTP request usando a API `fetch`, o JavaScript envia a request e continua executando outras instruções enquanto espera a resposta do server.

Além disso, frequentemente os recursos são alocados implicitamente por funções e métodos das APIs do browser ou do ambiente de execução. Por exemplo, ao criar um novo objeto `XMLHttpRequest` para fazer uma AJAX request, o browser aloca automaticamente os recursos necessários para tratar essa request.

No entanto, é importante ter em mente que, embora a alocação de recursos seja tratada automaticamente na maioria dos casos, ainda é responsabilidade do desenvolvedor garantir que esses recursos sejam liberados quando não forem mais necessários. Por exemplo, é importante fechar conexões de banco de dados ou liberar memória alocada por objetos grandes quando não estão mais em uso.

Para garantir um bom gerenciamento de recursos no JavaScript, é essencial seguir algumas boas práticas:

- Liberação Explícita de Recursos: sempre que possível, libere recursos explicitamente quando não forem mais necessários. Isso pode incluir fechar conexões ou liberar memória alocada por objetos grandes.
- Use Patterns Assíncronos: ao lidar com operações que envolvem recursos externos, como operações de input/output (I/O), use patterns assíncronos para garantir que o código continue executando com eficiência enquanto espera o recurso ficar disponível.
- Monitore o Uso de Recursos: em aplicações que lidam com um grande volume de recursos, é importante monitorar o uso de recursos para identificar potenciais gargalos ou leaks. Isso pode ser feito usando ferramentas de monitoramento ou profiling.
- Evite Resource Leaks: assim como memory leaks, resource leaks podem ocorrer se os recursos não forem devidamente liberados quando não forem mais necessários. Garanta fechar conexões e liberar recursos adequadamente para evitar leaks.

### Tratamento de Exceções

No JavaScript, exceções são tratadas através de um mecanismo de controle de fluxo chamado "exception handling". Quando um erro ocorre durante a execução do código, uma exceção é disparada, interrompendo o fluxo normal do programa. Código que pode potencialmente disparar uma exceção é colocado dentro de um bloco `try`, e o código que trata a exceção é colocado dentro de um bloco `catch`.

O mecanismo de exception handling no JavaScript é baseado em blocos `try...catch`, que permitem ao código tratar exceções que possam ocorrer durante a execução de forma controlada. Aqui está um exemplo simples de como funciona:

```js
try { // Código que pode disparar uma exceção throw new Error('Oops! Something went wrong.');} catch (error) { // Código que trata a exceção console.error('An error occurred:', error.message);}
```

Neste exemplo, o código dentro do bloco `try` é executado normalmente. Se uma exceção é disparada durante a execução desse código, o fluxo de execução é interrompido, e o controle é transferido para o bloco `catch`, onde o código pode tratar a exceção de forma controlada.

Além do mecanismo de exception handling baseado em blocos `try...catch`, o JavaScript também oferece a capacidade de disparar exceções manualmente usando a keyword `throw`. Isso permite aos desenvolvedores criar e disparar exceções customizadas em situações específicas.
